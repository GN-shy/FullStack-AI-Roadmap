# 多表查询
## 多表关系
表之间的三种关联关系：
项目开发设计数据库表时，业务模块存在关联，表与表之间分为三类关系：
1. 一对多（多对一）
2. 多对多
3. 一对一

### 一对多（多对一）
概念
一张主表（父表）的数据，可以对应子表多条记录；反过来子表多条记录归属同一条主表数据。
例：部门 和 员工、班级 和 学生、分类 和 商品。

设计方案
**在多的一方（子表）添加外键**，关联一的一方主键。
举例：
部门表(主表)：id(主键)、部门名称
员工表(子表)：id、姓名、**dept_id(外键，关联部门.id)**
一个部门有多名员工，一名员工只属于一个部门。

核心特点
1. 外键建在“多”的那张表；
2. 业务场景绝大多数都是一对多；
3. 可搭配外键删除行为（CASCADE/SET NULL等）控制联动操作。

### 一对一
概念
A表一条记录，仅对应B表一条记录，双向唯一绑定。

两种设计方案
1. **方案1（推荐）**：任意一张表新增外键，同时给外键加`UNIQUE`唯一约束
   例：用户表 和 用户详情表
   用户表：id(主键)、姓名、手机号
   用户详情表：id、身高、住址、**user_id(外键+UNIQUE)**
   一个用户只有一份详情，一份详情只属于一个用户。
2. **方案2**：两张表主键完全一致，详情表主键同时作为外键关联主表主键。

适用场景
1. 表字段过多，拆分冷热数据（用户基础信息/用户私密拓展信息）；
2. 权限隔离，部分敏感数据单独分表存储。

核心特点
区别于一对多：外键必须加唯一约束，保证只能对应一条数据。

### 多对多
概念
A表一条数据对应B表多条，B表一条也对应A表多条。
例：学生 & 课程、订单 & 商品、角色 & 菜单权限。

设计方案
**必须新建一张中间关联表**，中间表分别存储两张主表的外键。
举例：学生 与 课程
1. 学生表：sid(主键)、学生姓名
2. 课程表：cid(主键)、课程名
3. 中间表 student_course：
   - sid（外键关联学生表）
   - cid（外键关联课程表）
   - 联合主键：sid + cid，防止重复选课

一名学生可选多门课，一门课被多名学生选择，依靠中间表建立双向多对多关系。

核心特点
1. 不能直接在原有两张表加字段实现，必须创建中间表；
2. 中间表可额外拓展业务字段（如选课分数、下单数量）；
3. 中间表两个外键通常组合为联合唯一/联合主键。

## 多表查询基础概念
### 1. 多表查询概述
多表查询指**同时从多张数据表中联合检索数据**，适用于业务数据分散在不同关联表的场景（如同时查询员工表、部门表获取员工所属部门信息）。

### 2. 笛卡尔积（笛卡尔乘积）
#### 定义
数学概念：集合A与集合B中所有元素的无序组合全集。
放到数据库中：两张表直接关联查询、不加匹配条件时，会生成**表A每条记录 × 表B每条记录**的全部组合结果。

#### 示例说明
- 员工表共10条数据，部门表共3条数据
- 无匹配条件直接查询会产生 `10 × 3 = 30` 条组合记录
- 其中绝大多数组合无业务逻辑，属于**无效笛卡尔积**

#### 核心注意点
多表查询必须添加**表关联匹配条件**（通常用外键相等），过滤掉无意义的笛卡尔积组合，只保留符合业务关联的数据。
示例过滤条件：`emp.dept_id = dept.id`

#### 错误示范（产生笛卡尔积）
```sql
SELECT * FROM emp, dept;
```

#### 正确示范（消除无效笛卡尔积）
```sql
SELECT * FROM emp, dept WHERE emp.dept_id = dept.id;
```

# 多表查询分类
## 连接查询
1. 内连接：相当于查询A、B交集部分数据
2. 外连接：
   - 左外连接：查询左表所有数据，以及两张表交集部分数据
   - 右外连接：查询右表所有数据，以及两张表交集部分数据
3. 自连接：当前表与自身的连接查询，自连接必须使用表别名

## 子查询
将查询结果作为另一条 SQL 语句的查询条件、数据源或常量，嵌套在 SELECT/INSERT/UPDATE/DELETE 语句内部的查询语句，也叫内部查询；外层语句称为主查询。

## 内连接
作用：只查询两张表能匹配上的交集数据，分为隐式内连接、显式内连接。
### 隐式内连接
语法
```sql
SELECT 字段列表 FROM 表1 , 表2 WHERE 关联条件 ... ;
```
特点
多张表使用逗号分隔写在`FROM`后，表与表的关联条件、业务过滤条件全部写在`WHERE`子句。

### 显式内连接
语法
```sql
SELECT 字段列表 FROM 表1 [INNER] JOIN 表2 ON 关联条件 ... ;
```
特点
使用 `INNER JOIN` 关键字关联两张表，**表关联条件单独写在ON后**，业务过滤条件统一放在`WHERE`后。

### 隐式内连接 vs 显式内连接区别
1. 可读性
   - 隐式：关联条件、过滤条件混在 WHERE 中，多表联查时代码混乱，难以区分表关联逻辑；
   - 显式：`ON`专门存放表关联关系，WHERE 只做业务筛选，层次清晰，便于维护。
2. 扩展性
   - 隐式语法无法直接改为左 / 右外连接，需要完全重写；
   - 显式仅需修改 `INNER JOIN` 为 `LEFT JOIN` / `RIGHT JOIN`，改动极小。
3. 安全性
   - 隐式容易漏写关联条件，直接生成海量笛卡尔积；
   - 显式语法强制书写`ON`关联条件，减少失误。
4. 性能
   MySQL 底层优化器执行计划完全一致，查询效率无差异。
5. 开发规范
   企业项目推荐统一使用显式内连接，隐式仅用于简单临时查询。

## 外连接
### 左外连接
```sql
SELECT 字段列表 FROM 表1 LEFT [OUTER] JOIN 表2 ON 条件 ... ;
```
说明：相当于查询表1(左表)的所有数据，包含表1和表2交集部分的数据

### 右外连接
```sql
SELECT 字段列表 FROM 表1 RIGHT [OUTER] JOIN 表2 ON 条件 ... ;
```
说明：相当于查询表2(右表)的所有数据，包含表1和表2交集部分的数据

## 自连接
### 自连接查询语法
```sql
SELECT 字段列表 FROM 表A 别名A JOIN 表A 别名B ON 条件 ... ;
```
说明：自连接查询，可以是内连接查询，也可以是外连接查询。

## 联合查询—union union all
# UNION 查询
概念
union查询，就是把多次查询的结果合并起来，形成一个新的查询结果集。

语法
```sql
SELECT 字段列表 FROM 表A ...
UNION [ALL]
SELECT 字段列表 FROM 表B ...;
```

补充区分
1. `UNION`：合并结果并自动去重；
2. `UNION ALL`：直接合并全部结果，不去重，性能更高；
3. 要求：前后两条SELECT查询的**字段数量、字段类型必须一致**。

## 子查询
概念：SQL语句中嵌套SELECT语句，称为嵌套查询，又称子查询。
```sql
SELECT * FROM t1 WHERE column1 = ( SELECT column1 FROM t2 );
```
子查询外部的语句可以是INSERT / UPDATE / DELETE / SELECT 的任何一个。
根据子查询结果不同，分为：
标量子查询（子查询结果为单个值）
列子查询(子查询结果为一列)
行子查询(子查询结果为一行)
表子查询(子查询结果为多行多列)

### 标量子查询（子查询结果为单个值，一行一列）
子查询执行后只返回一个数值，可使用 = > < >= <= != 等普通比较运算符匹配。
语法格式：
```sql
SELECT 字段 FROM 表 WHERE 字段 = (标量子查询);
```
示例：查询市场部所有员工，先单独查出市场部的部门id作为固定值
```sql
SELECT * FROM emp WHERE dept_id = (SELECT id FROM dept WHERE dept_name = '市场部');
```
注意：子查询必须仅返回一条一条数据，若返回多行则直接报错。

### 列子查询（子查询结果为一列，多行一列）
子查询返回一整列多个值，不能直接用普通等号，搭配 IN / ANY / SOME / ALL 使用。
- IN：匹配子查询结果中任意一个值
- ANY/SOME：满足和子查询任意一条数据的比较条件
- ALL：满足和子查询全部数据的比较条件
  语法格式：
```sql
SELECT 字段 FROM 表 WHERE 字段 IN (列子查询);
```
示例：查询存在员工的所有部门id与部门名称
```sql
SELECT * FROM dept WHERE id IN (SELECT DISTINCT dept_id FROM emp);
```

### 行子查询（子查询结果为一行，一行多列）
子查询返回单行多列数据，使用 (值1,值2) 行构造符，同时匹配多个字段。
语法格式：
```sql
SELECT 字段 FROM 表 WHERE (字段1,字段2) = (行子查询);
```
示例：查询工资和入职时间与张三完全相同的员工
```sql
SELECT * FROM emp WHERE (salary, entry_date) = (SELECT salary, entry_date FROM emp WHERE name = '张三');
```

### 表子查询（子查询结果为多行多列，完整临时表）
子查询返回完整二维数据集，当作一张临时表使用，一般放在FROM后作为派生表，必须给子查询起别名。
语法格式：
```sql
SELECT 字段 FROM (表子查询) 临时表别名 WHERE 条件;
```
示例：统计每个部门平均工资，筛选平均工资大于6000的部门
```sql
SELECT * FROM (SELECT dept_id, AVG(salary) avg_sal FROM emp GROUP BY dept_id) t WHERE t.avg_sal > 6000;
```
