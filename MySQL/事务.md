# 事务
## 一、事务简介
事务是一组不可分割的DML（INSERT/UPDATE/DELETE）SQL操作单元，这一组操作要么全部执行成功，要么全部执行失败回滚，不会出现部分成功、部分失败的中间状态。
典型场景：转账操作，A扣款、B收款必须同时生效，若中途程序崩溃，两条SQL全部撤销。
MySQL中InnoDB引擎支持事务，MyISAM不支持事务。

## 二、事务操作
### 1. 基础手动事务指令
```sql
-- 1. 开启事务
START TRANSACTION;

-- 执行多条增删改SQL
UPDATE account SET money = money - 100 WHERE id = 1;
UPDATE account SET money = money + 100 WHERE id = 2;

-- 2. 提交事务：永久保存修改，结束事务
COMMIT;

-- 3. 回滚事务：撤销本次事务所有修改，结束事务
ROLLBACK;
```

### 2. 自动提交机制
MySQL默认自动提交事务，执行一条DML语句立刻自动commit。
```sql
-- 查看自动提交状态，1开启，0关闭
SELECT @@autocommit;

-- 关闭自动提交（当前会话生效）
SET autocommit = 0;
```

### 3. 保存点（局部回滚）
```sql
START TRANSACTION;
INSERT INTO account VALUES(3,'李四',1000);
-- 创建保存点s1
SAVEPOINT s1;
UPDATE account SET money=0 WHERE id=3;
-- 仅回滚到保存点s1，不会撤销之前的insert
ROLLBACK TO s1;
COMMIT;
```

## 三、事务四大特性 ACID
1. **原子性 Atomicity**
  事务是最小执行单元，内部操作不可分割，全部成功或全部失败回滚。底层依靠undo日志实现。
2. **一致性 Consistency**
  事务执行前后，数据的业务约束完整不变（转账前后总金额不变），原子性、隔离性、持久性共同保障一致性。
3. **隔离性 Isolation**
  多个事务并发执行时，事务之间互相隔离，互不干扰。底层依靠undo、redo、锁机制实现。
4. **持久性 Durability**
  事务提交后，对数据的修改永久保存，数据库宕机、断电数据不会丢失。底层依靠redo日志实现。

## 四、并发事务问题
多个事务同时操作同一张表数据，会产生三类并发问题：
1. **脏读**
  一个事务读取到另一个事务**未提交**的修改数据，对方回滚后，读到的数据失效。
2. **不可重复读**
  同一个事务内，先后两次读取同一行数据，中间被其他事务提交更新，两次查询结果不一致。侧重：update修改。
3. **幻读**
  同一个事务内，多次按条件查询数据，中间被其他事务提交新增/删除，前后查询的行数发生变化。侧重：insert/delete新增消失数据。

## 五、事务隔离级别
### 1. 四种隔离级别（从低到高）
| 隔离级别                                     | 脏读   | 不可重复读 | 幻读   | 性能 |
| -------------------------------------------- | ------ | ---------- | ------ | ---- |
| 读未提交 READ UNCOMMITTED                    | 存在   | 存在       | 存在   | 最高 |
| 读已提交 READ COMMITTED（Oracle默认）        | 不存在 | 存在       | 存在   | 较高 |
| 可重复读 REPEATABLE READ（MySQL InnoDB默认） | 不存在 | 不存在     | 存在   | 中等 |
| 串行化 SERIALIZABLE                          | 不存在 | 不存在     | 不存在 | 最低 |

### 2. 隔离级别操作语法
```sql
-- 查看当前会话隔离级别
SELECT @@transaction_isolation;

-- 设置会话隔离级别
SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED;

-- 设置全局隔离级别（重启数据库永久生效）
SET GLOBAL TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

### 3. 级别说明
1. READ UNCOMMITTED：能读到其他事务未提交数据，生产环境不使用；
2. READ COMMITTED：只能读取已提交数据，解决脏读，互联网项目常用；
3. REPEATABLE READ：可重复读，MySQL默认，解决脏读、不可重复读；
4. SERIALIZABLE：最高隔离，事务串行执行，完全无并发问题，并发量高场景会大量锁等待，极少使用。
