

# 阻止默认行为与阻止冒泡
## 一、阻止默认行为
### 概念
某些情况下需要阻止浏览器自带的默认行为，比如阻止链接跳转、表单提交跳转、右键菜单、文本拖拽等。
### 语法
```js
e.preventDefault()
```
### 案例：阻止表单默认提交
```html
<form action="http://www.baidu.com">
  <input type="submit" value="提交">
</form>
<script>
  const form = document.querySelector('form')
  form.addEventListener('click', function (e) {
    // 阻止表单默认提交跳转行为
    e.preventDefault()
  })
</script>
```

## 二、阻止事件冒泡
### 概念
事件冒泡：子元素触发事件后，事件会逐层向上传递给父级、祖先级元素。
**阻止冒泡**：让事件只在当前元素执行，不再向上传递给父元素。
### 语法
```js
e.stopPropagation()
```
### 完整案例（嵌套结构演示）
```html
<div class="father">
  父盒子
  <div class="son">子盒子</div>
</div>
<script>
  const father = document.querySelector('.father')
  const son = document.querySelector('.son')

  // 父元素绑定点击事件
  father.addEventListener('click', function () {
    alert('我是父盒子')
  })
  // 子元素绑定点击事件，同时阻止冒泡
  son.addEventListener('click', function (e) {
    alert('我是子盒子')
    // 阻止事件向上冒泡到父元素
    e.stopPropagation()
  })
</script>
```
> 效果：点击子盒子，只会触发子盒子弹窗，不会触发父盒子弹窗。

---

## 三、核心区别总结
1. **`e.preventDefault()`**：阻止**浏览器默认行为**（表单提交、a标签跳转等），不影响事件冒泡。
2. **`e.stopPropagation()`**：阻止**事件向上冒泡**，不影响默认行为。
3. 可同时使用：
```js
e.preventDefault()  // 阻止默认行为
e.stopPropagation() // 阻止冒泡
```
