

# 事件委托
事件委托是利用事件流的特征解决一些开发需求的知识技巧

## 优点
减少注册次数，可以提高程序性能；动态新增的子元素无需重新绑定事件，简化代码维护

## 原理
事件委托其实是利用**事件冒泡**的特点，给父元素注册事件，当我们触发子元素的时候，会冒泡到父元素身上，从而触发父元素的事件

## 实现
事件对象`e.target.tagName`可以获得真正触发事件的元素

---

## 完整代码案例
### 案例1：列表点击事件委托（最常用）
```html
<ul id="list">
  <li>选项1</li>
  <li>选项2</li>
  <li>选项3</li>
</ul>
<script>
  // 给父元素ul绑定事件，而不是给每个li绑定
  const ul = document.getElementById('list')
  ul.addEventListener('click', function(e) {
    // 获取真正点击的子元素
    const target = e.target
    // 判断点击的是否为li标签
    if (target.tagName === 'LI') {
      console.log('你点击了：', target.innerText)
    }
  })
</script>
```

### 案例2：动态新增元素也能触发
事件委托最大优势：**后续JS动态添加的子元素，自动继承事件，无需重复绑定**
```html
<ul id="list2">
  <li>原有选项</li>
</ul>
<button id="addBtn">新增选项</button>
<script>
  const ul = document.getElementById('list2')
  const addBtn = document.getElementById('addBtn')

  // 父元素委托事件
  ul.addEventListener('click', function(e) {
    if (e.target.tagName === 'LI') {
      alert(`选中：${e.target.textContent}`)
    }
  })

  // 动态新增li
  addBtn.onclick = function() {
    const li = document.createElement('li')
    li.innerText = '新选项'
    ul.appendChild(li)
  }
</script>
```

## 关键知识点补充
1. **事件对象属性区分**
   - `e.target`：真正触发事件的元素（实际点击的子元素）
   - `e.currentTarget`：绑定事件的元素（父元素，固定不变）
2. **适用场景**
   - 大量同类子元素需要绑定相同事件（列表、按钮组、导航）
   - 子元素会动态新增/删除
3. **不适用场景**
   - 事件不会冒泡：`blur`、`focus`、`load`等，无法使用事件委托
   - 需要精准阻止冒泡、或子元素事件逻辑差异极大时
4. **优化写法**
   除了判断`tagName`，也可以用**类名判断**，更灵活：
   ```js
   if (e.target.classList.contains('item')) {
     // 点击类名为item的元素触发
   }
   ```

## 核心总结
- 核心：**父绑定，子触发，靠冒泡**
- 好处：性能高、代码少、支持动态元素
- 关键：通过`e.target`判断实际触发元素，避免父元素自身触发事件
