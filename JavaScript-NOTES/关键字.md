# 关键字

## this关键字

### 可以解决的问题

局部变量与成员变量重名时的就近原则

```java
public class Student {
    // 成员变量
    String name;
    int age;

    public void sayHello(String name) {
        // 局部变量 name 与成员变量 name 重名
        // Java 遵循「就近原则」，此处使用的是方法参数里的局部变量 name
        System.out.println(name);
    }
}
```

- **就近原则**：当局部变量和成员变量同名时，Java 会优先使用**离当前代码更近的变量**（即局部变量）。
- **访问成员变量**：如果想在重名时访问成员变量，需要使用 `this.变量名` 来明确指代：
  ```java
  public void sayHello(String name) {
      // 局部变量
      System.out.println(name);
      // 成员变量（通过 this 访问）
      System.out.println(this.name);
  }
  ```



### 作用

调用本类的成员：this.本类成员变量；

​                         this.本类成员方法（）；

```
package com.itheima.mthis;

public class Person {
    // 成员变量
    int num = 10;

    public void show() {
        // 局部变量（与成员变量同名）
        int num = 20;
        // 就近原则：直接访问局部变量
        System.out.println(num);    // 输出 20
        // this.num：明确访问成员变量
        System.out.println(this.num); // 输出 10
    }

    public void print() {
        System.out.println("Print...");
    }

    public void method() {
        // 访问成员变量（无同名局部变量时，this 可省略）
        System.out.println(this.num); // 输出 10
        // 调用本类其他方法（this 可省略）
        this.print(); // 输出 "Print..."
    }
}
```

***省略规则*** ：this.本类成员变量：如果不涉及重名问题，this.可以省略不写

### this介绍

- this代表当前类对象的引用（地址）
- 哪一个对象调用的方法，方法中的this就代表哪一个对象
