# 页面加载事件 元素滚动事件 页面尺寸事件

## 页面加载事件

> load 事件
>
> 加载外部资源（如图片，外联CSS和JavaScript等）加载完毕时触发的事件
>
> 使用场景：
> 1. 需要等页面资源全部处理完后执行某些操作
> 2. 需要获取图片宽高、样式计算结果等依赖资源加载完成的场景
>
> 事件名：load
>
> 监听页面所有资源加载完毕：
> 给window添加load事件
> ```javascript
> // 页面加载事件（等待页面所有资源加载完毕，就会去执行回调函数）
> window.addEventListener('load', function () {
>   // 执行的操作
> })
> ```
>



> DOMContentLoaded 事件
>
> 当初始的HTML文档被完全加载和解析完成之后，DOMContentLoaded事件被触发，而无需等待样式表，图像等完全加载
>
> 使用场景：
> 1. 老代码把script写在head中，需要安全获取DOM元素
> 2. 只需操作DOM结构，不需要等待图片等资源加载
>
> 事件名：DOMContentLoaded
>
> 监听页面DOM加载完毕：
> 给document添加DOMContentLoaded事件
> ```javascript
> document.addEventListener('DOMContentLoaded', function () {
>   // 执行的操作
> })
> ```
>



> 事件对比
>
> - 触发时机：load等待页面所有资源加载完成，DOMContentLoaded仅等待HTML解析和DOM树构建完成
>
> - 执行速度：DOMContentLoaded更快，load相对较慢
>
> - 绑定对象：load绑定window，DOMContentLoaded绑定document
>
> - 适用场景：load适用于依赖资源加载完成的操作，DOMContentLoaded适用于DOM操作
>
> - # 什么时候用 `DOMContentLoaded`
>
>   使用场景：
>
>   1. **你要获取页面元素**（按钮、div、input、p 等标签）
>   2. **script 写在 head 里**，怕找不到元素
>   3. **只想操作 DOM，不需要等图片、视频加载**
>   4. **希望页面一解析完就立刻执行 JS**
>
>   **只要是操作页面元素 → 一律用 DOMContentLoaded**
>
>   ------
>
>   # 什么时候用 `load`
>
>   使用场景：
>
>   1. **需要获取图片的宽高**
>   2. **需要等图片、样式、视频全部加载完**
>   3. **需要计算元素最终样式**
>   4. **页面所有资源都加载完再执行**
>
>   **必须等图片 / 资源加载完才能做的事 → 才用 load**
>
>   ------
>
>   # 总结
>
>   - **操作元素、找 DOM → DOMContentLoaded**
>   - **等图片、等所有资源加载完 → load**
>
>





## 元素滚动事件

滚动条在滚动的时候持续触发的事件

使用场景：很多网页需要检测用户把页面滚动到某个区域后做一些处理，比如固定导航栏，比如返回顶部

事件名：scroll

监听整个页面滚动

```
// 页面滚动事件
window.addEventListener('scroll', function () {
  // 执行的操作
})
```

# scrollLeft和scrollTop（属性）
- 获取被卷去的大小
- 获取元素内容往左、往上滚出去看不到的距离
- 这两个值是可读写的

尽量在scroll事件里面获取被卷去的距离

```
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>scroll 事件获取 scrollTop/scrollLeft</title>
  <style>
    .box {
      width: 300px;
      height: 300px;
      border: 2px solid #333;
      /* 必须设置，才能出现滚动条 */
      overflow: auto;
    }
    .content {
      width: 800px;
      height: 1000px;
    }
  </style>
</head>
<body>
  <div class="box">
    <div class="content">内容区域</div>
  </div>

  <script>
    const box = document.querySelector('.box')

    // 在 scroll 事件里面获取被卷去的距离（标准写法）
    box.addEventListener('scroll', function () {
      // 垂直被卷去的距离
      console.log('scrollTop：', box.scrollTop)
      // 水平被卷去的距离
      console.log('scrollLeft：', box.scrollLeft)
    })
  </script>
</body>
</html>
```



## 页面尺寸事件

# 元素尺寸、位置 & 窗口 resize 事件
一、获取元素宽高

1. offsetWidth / offsetHeight

- 获取元素**自身可见宽高**，包含：`width/height + padding + border`
- 返回纯数字，方便直接计算
- 注意：盒子隐藏（`display: none`）时，获取结果为 `0`

2. clientWidth / clientHeight

- 获取元素**内容区可见宽高**，包含：`width/height + padding`
- 不包含：边框、margin、滚动条

二、窗口 resize 事件

窗口尺寸改变时触发
```javascript
window.addEventListener('resize',function(){
    // 执行的代码
})
```

三、代码示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>offset与client获取宽高</title>
  <style>
    .box {
      width: 200px;
      height: 200px;
      padding: 20px;
      border: 5px solid #000;
      background: #ccc;
    }
  </style>
</head>
<body>
  <div class="box"></div>

  <script>
    const box = document.querySelector('.box')
    // 获取offset宽高（包含border+padding）
    console.log('offsetWidth：', box.offsetWidth)
    // 获取client宽高（包含padding，不含border）
    console.log('clientWidth：', box.clientWidth)

    // 窗口大小改变时触发
    window.addEventListener('resize', function () {
      console.log('窗口大小改变')
    })
  </script>
</body>
</html>
```
