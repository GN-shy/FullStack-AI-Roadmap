# 本地存储

## 本地存储介绍

- 问题：原生页面临时数据刷新页面即丢失
- 背景：网页应用日趋复杂，HTML5 推出浏览器本地存储方案，解决本地持久化存数据需求

### 本地存储三大特点

1. 数据存储在**用户浏览器**内
2. 读写便捷，页面刷新数据不会丢失
3. 容量较大，sessionStorage和localStorage约5M左右





## 本地存储分类

### localStorage

**作用**：可以将数据永久存储在本地 (用户的电脑)，除非手动删除，否则关闭页面也会存在

特性

- 可以多窗口（页面）共享（同一浏览器可以共享）
- 以键值对的形式存储使用

语法

存储数据：

```javascript
localStorage.setItem(key, value)
```

获取数据：

```javascript
localStorage.getItem(key)
```
- `key`：存入数据时设置的键名，和`setItem`的key保持一致
- 返回值：对应key存储的value值，无数据则返回`null`

删除数据

```
localStorage.removeItem(key)
```

改：

```
localStorage.setItem(key,新值)
```



### sessionStorage

特性：

- 生命周期为关闭浏览器窗口
- 在同一个窗口(页面)下数据可以共享
- 以键值对的形式存储使用
- 用法跟localStorage 基本相同

配套API

```js
// 存数据
sessionStorage.setItem(key,value)
// 取数据
sessionStorage.getItem(key)
// 删除单条
sessionStorage.removeItem(key)
// 清空全部
sessionStorage.clear()
```



## 存储复杂数据类型

- 要点：`localStorage`、`sessionStorage`**只能存储字符串**，直接存入对象/数组会转为`[object Object]`丢失数据
```js
const goods = {
  name: '小米10',
  price: 1999
}
// 错误写法
localStorage.setItem('goods', goods)
```
- **解决**：需要将复杂数据类型转换成 JSON 字符串，再存储到本地
- **语法**：`JSON.stringify(复杂数据类型)`

```
// 存储示例
const goods = {name:'小米10',price:1999}
localStorage.setItem('goods',JSON.stringify(goods))

// 配套读取：JSON.parse()把JSON字符串还原为对象
const res = JSON.parse(localStorage.getItem('goods'))
```

```
// json字符串 → js对象：JSON.parse()
let jsonStr = '{"name":"小米10","price":1999}'
let obj = JSON.parse(jsonStr)

console.log(obj.name) // 小米10
```
