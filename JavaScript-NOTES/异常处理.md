# 异常处理

JavaScript 提供了多种异常处理方式，用来捕获、抛出和调试错误，提升代码运行的健壮性。

---

## 1. throw：主动抛出异常

- **作用**：手动抛出一个自定义错误，中断后续代码执行，常用于主动校验失败时抛出异常。
- **语法**：
```javascript
throw 错误信息
// 可以抛出字符串、数字、布尔值，也可以抛出 Error 对象（推荐）
throw new Error('错误信息')
```

- **示例**：
```javascript
function checkAge(age) {
  if (age < 18) {
    // 主动抛出异常，阻止函数继续执行
    throw new Error('年龄必须大于等于18岁')
  }
  return '成年'
}

// 调用函数时，若传入不符合条件的值，会直接抛出错误
checkAge(16)
```

---

## 2. try / catch：捕获并处理异常

- **作用**：捕获代码运行时发生的错误，避免程序直接崩溃，还可以配合 `finally` 执行无论是否出错都要运行的代码。
- **语法结构**：
```javascript
try {
  // 可能出错的代码
} catch (err) {
  // 捕获错误，err 是错误对象，包含错误信息
} finally {
  // 无论 try 中是否出错，都会执行的代码（可选）
}
```

- **示例**：
```javascript
function parseJSON(str) {
  try {
    // 尝试解析 JSON 字符串
    const data = JSON.parse(str)
    console.log('解析成功', data)
    return data
  } catch (err) {
    // 捕获解析失败的错误
    console.error('解析失败，错误信息：', err.message)
    // 可以给默认值，避免后续代码报错
    return {}
  } finally {
    console.log('解析操作结束')
  }
}

// 正确的 JSON 字符串
parseJSON('{"name":"佩奇"}')
// 错误的 JSON 字符串
parseJSON('{name:"佩奇"}')
```

## 3. debugger：调试断点

- **作用**：在代码中插入断点，当浏览器开发者工具或 Node.js 调试器开启时，代码会在 `debugger` 处暂停，方便逐行查看变量和执行流程。
- **语法**：
```javascript
debugger
```

- **示例**：
```javascript
function add(a, b) {
  debugger // 调试器会在这里暂停
  const sum = a + b
  return sum
}

add(1, 2)
```

> 注：`debugger` 仅在调试工具开启时生效，生产环境建议移除。

## 4. 综合使用示例

```javascript
function divide(a, b) {
  try {
    debugger // 调试时可在此处查看参数
    if (b === 0) {
      throw new Error('除数不能为0')
    }
    return a / b
  } catch (err) {
    console.error('出错了：', err.message)
    return null
  }
}

console.log(divide(10, 2)) // 5
console.log(divide(10, 0)) // 出错了：除数不能为0 → null
```

## 5. 补充知识点

### 5.1 常见的内置错误类型
- `Error`：通用错误类型
- `SyntaxError`：语法错误
- `ReferenceError`：引用错误（使用了未定义的变量）
- `TypeError`：类型错误（对非预期类型执行操作）
- `RangeError`：范围错误（数值超出有效范围）

### 5.2 `try/catch` 的使用场景
- 解析 JSON 数据
- 网络请求
- 用户输入校验
- 文件读取操作（Node.js）

### 5.3 错误对象的常用属性
- `err.message`：错误描述信息
- `err.name`：错误类型名称
- `err.stack`：错误调用栈信息（调试用）
